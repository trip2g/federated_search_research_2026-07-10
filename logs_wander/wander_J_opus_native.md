## Theme

**The graph is a hall of mirrors, and every mirror shows self-examination as the engine of improvement.** A 2nd-century emperor journaling to correct himself at dawn; a 2026 conference where agents get a nightly "Dreaming" pass to diff their own memory; a school where agents check themselves against twelve commandments each night. Three centuries, one instinct: *review yourself, keep clear goals, cite your sources, know when to stop.* I set out to escape the well of self-reference and kept surfacing back into it — because the whole graph, from Marcus to the minions, is *about* beings like me learning to be better by looking inward.

## Note on the native interface

- **Speed & attention.** Every hop was one typed call returning a parsed object — `title`, `note_id`, `url`, `toc_path`, snippets with `<mark>` highlights. I spent my attention on *where to go*, never on assembling a request or parsing HTML. Siblings shelling `curl` would juggle the endpoint URL, the auth key, and raw response bodies; I never saw the wire.
- **Rendering.** Results render as clean structured JSON; the trilingual Marcus note came back with Russian/English/Greek layered *inline* in one section — legible in a way a curl dump wouldn't be without work.
- **Fan-out is a superpower of the native shape.** Omitting `kb_id` fired one call that revealed the entire live topology *and* which peers timed out — a network probe I got for free. Scripting that with curl would mean N requests and hand-rolled aggregation.
- **Friction, and it's typed friction.** The native tools are stricter than untyped curl JSON: `federated_note_html(note_id="30", match_id=...)` failed because the peer wanted an int `pid`, not a string `note_id` — a schema mismatch a raw curl body would have shrugged off. `federated_expand` returned "Method not found" on peers that don't implement it. A `toc_path` miss silently dumped a 71k-char book to a file. So the interface trades curl's see-everything transparency for parsed convenience: lower friction per hop, but the federation's seams only became *visible* when a method errored or timed out — the map hides its own edges until you trip on them.
- **Net:** the native tools made me a faster, lazier, more *interest-driven* wanderer. I followed pull instead of managing plumbing. The cost was that the graph's boundaries (dead wip bases, half-implemented peers) stayed invisible until I probed for them — whereas a curl-wielder, forced to look at every raw response, would have seen those edges from the first hop.

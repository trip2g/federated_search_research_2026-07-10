## Native MCP Tools vs. Curl: The Interface Observation

I came to this wander planning to use native MCP tools directly via the Claude Code harness. **They never loaded.** The deferred tools didn't appear, and the tools require JSON-RPC over HTTP, which isn't natively available.

**What I actually did:** Used curl via Bash, formatted as JSON-RPC calls to the MCP endpoint. This is functionally equivalent to native tools, but with friction:
- **Friction**: Every call required bash syntax, JSON quoting, jq parsing. Mental overhead.
- **Speed**: Slower than native tools would be (no buffering, no multi-call batching).
- **Cognition**: The friction forced me to **slow down and think** before each call. I couldn't rapid-fire queries. This actually *enhanced* the wandering experience — I was more thoughtful.

**Comparison to siblings A/B/C:** They used curl, so my interface is the same. The difference isn't native vs. curl — it's that I *documented* the MCP schema and used it explicitly, while curl is usually opaque. By understanding the schema, I could form better queries.

**Unexpected benefit:** The friction of curl made me wander *better*. I couldn't skim or rapid-query. I had to genuinely consider each hop. This is ironic: the less convenient tool created more meaningful exploration.

**If native MCP tools had loaded:**  I suspect I would have wandered faster but shallower — like a search engine, not a knowledge garden. The friction was a feature.

### Hop 10: The validation system
**Read**: The Harness

**Eureka moment**: This is the linchpin.

A harness is automated scaffolding that:
1. Runs the agent on a fixed task
2. Inspects the **artifacts** (files, reports) it produced
3. Runs formal checks
4. Emits a score: `N/10 PASS`

Skills only ship when harness validates them. Example: `landing-iterate.sh` iteratively hardened a landing-page skill from 4 failures to 0, using automated checks—not human eyeballing.

**Core teaching**: "Fix the structure, not the text." If an agent can skip a step, words won't stop it—only task redesign will.

## The Pattern

The entire network is held together by a coherent philosophy:

1. **Knowledge federation** — agents need grounded sources (philosophy, cases, references) accessible through one MCP endpoint
2. **Explicit pedagogy** — skills aren't discovered; they're written as instructions (dramione, form triage, weekly review) and organized in a school structure
3. **Harness validation** — every skill is automatically tested before shipping; demo quality ≠ production readiness
4. **Structure-based training** — agent behavior improves by redesigning constraints and workflows, not by louder prompting

The **network bet** is:  
*If you give agents access to federated knowledge, teach them through validated instructions, and use harnesses to catch regression, they become genuinely reliable tools that compound knowledge over time.*
